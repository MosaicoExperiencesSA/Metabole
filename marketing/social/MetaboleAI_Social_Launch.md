## 0. In due righe

Apriamo la presenza social di MetaboleAI su **6 piattaforme**, con un solo handle (`@metaboleai`), una sola voce e un motore di contenuti che **riusa quello che già produciamo** (vignette, diari del percorso, risultati clienti). L'obiettivo del lancio non è "fare like": è **alimentare la prova gratuita di 8 giorni** e dare alle coach materiale credibile da mostrare.

**La regola:** un contenuto si gira in 3-4 formati. Non produciamo 6 cose diverse al giorno — ne produciamo **una**, e la vestiamo per ogni piattaforma.

## 5. Spec integrazione dashboard — per Simone

> Obiettivo: dal **modulo Marketing → Social** della dashboard, pubblicare (o schedulare) un post su più piattaforme e leggere le metriche base, senza uscire dal backoffice. Serve a chiudere il cerchio: contenuto → pubblicazione → funnel del lancio.

### Realtà delle API (perché non sono tutte uguali)
Ogni piattaforma ha una sua API, con oneri di **app review** e permessi diversi. Costruire 6 connettori nativi è settimane di lavoro e manutenzione continua (i permessi Meta/TikTok cambiano spesso). Per questo la raccomandazione è a due strade.

**Strada A — Aggregatore (consigliata per partire).**
Usare un provider che espone **una sola API** verso tutte le piattaforme: **Ayrshare** o **Metricool** (entrambi coprono IG, FB, TikTok, YouTube, Pinterest, Threads; publishing + analytics + scheduling).
- **Pro:** un solo connettore, app review gestita da loro, live in **giorni non settimane**, meno manutenzione.
- **Contro:** costo mensile per seat/profilo, dipendenza da terzi.
- **MVP dashboard:** una tabella `SocialPost` (testo, media[], piattaforme[], `scheduledAt`, `status`, `externalId`) + un service che chiama l'aggregatore. Webhook per lo stato "pubblicato/fallito".

**Strada B — Connettori nativi (in un secondo momento, per togliere il costo del terzo).**
Solo se e quando i volumi giustificano il lavoro. Ordine di priorità = ordine di valore:

| Piattaforma | API | Permessi / note critiche |
|---|---|---|
| **Facebook + Instagram** | **Meta Graph API** | IG dev'essere *Business* e **collegato alla Pagina**. Permessi `instagram_content_publish`, `pages_manage_posts`. Richiede **App Review** + Business verification. IG: publishing solo di immagini/video (no Storie via API standard). **Fai questo per primo** — copre il 70% del valore. |
| **YouTube** | **YouTube Data API v3** | Upload video/Shorts. Quota giornaliera (un upload ≈ 1600 unità su 10.000/giorno). OAuth Google. |
| **TikTok** | **Content Posting API** | Da **TikTok for Developers**, richiede audit dell'app. In assenza di audit completo, pubblica come **bozza** da rifinire nell'app. |
| **Pinterest** | **Pinterest API v5** | Crea pin, richiede dominio verificato + app in produzione (review). |
| **Threads** | **Threads API** (Meta) | API ufficiale recente; publishing testo/media, legata all'account IG/Meta. |
| **WhatsApp Channel** | *nessuna API di publishing pubblica* | I **Canali** non hanno API di pubblicazione: si postano a mano dall'app. In dashboard mettere solo un promemoria/checklist, non un connettore. |

### Raccomandazione operativa
1. **MVP = Strada A** (aggregatore) con IG+FB+TikTok+Pinterest attivi da subito. Un solo service, una sola tabella, scheduling incluso.
2. Nel modulo dashboard: **compositore** (testo + upload media + selezione piattaforme + data/ora) → salva `SocialPost` → invia all'aggregatore → mostra stato.
3. **Analytics minimi** dal medesimo provider: follower, reach, engagement per post — così finiscono nello stesso *Funnel del lancio* dove già guardiamo prova→pagante.
4. WhatsApp Channel resta **manuale** (solo checklist in dashboard).
5. Quando i costi dell'aggregatore superano il valore, si migra **Meta prima** ai connettori nativi (Strada B), lasciando il resto sull'aggregatore.

### Sicurezza / conformità
- Token OAuth cifrati a riposo, refresh gestito lato server, **mai** nel frontend.
- I permessi di pubblicazione sono legati all'app Meta di **Mosaico Experiences SA** (business verification con i dati di Lugano).
- Log di ogni pubblicazione (chi, cosa, quando, esito) per audit.
